# 制种

## 重点注意

由于种子的信息哈希从 `info` 字段计算得来，  
而不同制种软件、不同 BT 标准的 `info` 字段内容都不一样，  
有些不规范的工具重编码后也会改变 `info` 字段，  
BT 是按信息哈希认种的，而不是按文件内容，  
即使内容完全相同，信息哈希不同也算不同种子，无法互通，

所以制种后的种子必须作为**唯一**的传播种子，  
**所有**做种者和公开发布渠道都应该使用这个种子文件，  
严禁中途修改、更换种子文件

## 工具选用

严禁使用 BitComet 等不规范制种工具

若无需自动化工具，手动制种使用 Libtorrent 2 版本的最新版 qBittorrent

## 种子格式选择

### 格式介绍

BT 种子有三种格式: v1 v2 混合

#### v1

v1 是最早的 BT 规范，信息哈希为 `info` 字段的 SHA-1

必要字段:

* `info -> pieces`
* `info -> files`（目录） or `info -> length`（单文件）

#### v2

v2 是新的 BT 规范，信息哈希为 `info` 字段的 SHA-256

必要字段:

* `piece layers`
* `info -> file tree`
* `info -> meta version`

#### 混合

混合种子格式是同时含有 v1 和 v2 的所有必要字段

### 格式选择

首先确定需要发布到哪些 BT 站

大部分站在解析 torrent 文件时解析不到 v1 必要字段就会报错终止，所以大部分站不支持 v2 种子

有些站甚至不支持混合种子

按所有需发布站最低支持的种子格式发布，v2 > 混合 > v1

# 做种

## 工具选用

任何现代化的支持 BT v2 的做种工具皆可，新手推荐使用 Libtorrent 2 版本的最新版 qBittorrent

## 做种设置

按自己设备环境自行设置上传限速

建议开启**超级做种**模式，这样可以节约做种上传量，并且可以在多人同时下载时均匀上传
